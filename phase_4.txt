Role: You are a DevSecOps Engineer.

Task: We are now moving to Phase 4 of the ISRM project: CI/CD Integration. I need you to automate the security scanning process we used in Phase 3 by integrating it into a CI/CD pipeline.

Requirements:
1. Write a pipeline configuration file. Please provide a GitHub Actions workflow (`.github/workflows/security.yml`) as it is the most straightforward for Python web apps, or a `Jenkinsfile` if you prefer.
2. The pipeline must automate the following stages:
   - Build/Checkout: Pull the application code.
   - Setup: Install Python and project dependencies.
   - SAST Scan: Run Bandit against the application code to find vulnerabilities.
   - DAST Scan: Run an OWASP ZAP baseline scan against the application.
   - Artifacts: Save and archive the vulnerability reports generated by Bandit and ZAP.
3. The pipeline should be configured to fail the build automatically if high-severity vulnerabilities are detected.

Deliverable:
Please provide the complete code for the CI/CD configuration file. Also, provide a brief, simple explanation of what each stage in the pipeline does. Do not attempt to run a Jenkins server or CI runner in the terminal; just provide the configuration scripts.
